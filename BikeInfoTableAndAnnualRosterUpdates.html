1. Introduction

Each year teams riders may retire, move teams or join MotoGP. The setup of these changes in done via a spreadsheet hosted on Google Drive and converted to a table kept in BikeInfoTable.cs.

2. BikeInfoTable

The BikeInfoTable controls which visual assets are loaded for bike and rider in the game. By changing the BikeInfo table we switch assets between one year and the next. Each year new assets are created for in game ui images of bikes and riders and in race textures for the bikes and riders. All these are controlled by the BikeInfoTable.

The table is maintained as a google spreadsheet at https://docs.google.com/spreadsheets/d/1JSdhZVxPjCn8USXJu_WzOL4-jZ07-U80Tz7RxIks9EM/edit#gid=776858588

Each year we make a new tab containing the updated data. New annual assets are created with a two digit year prefix, e.g. 21_93_MARQUEZ* are the assets for 2021 for rider Marc Marquez #93.

All edits are made here in the speadsheet and then a tool is used to extract the data and format the data for inclusion in the file BikeInfoTable.cs

Below is a description of the meaning of each of the columns in the table:

column in spreadsheet meaning
id This is the id of the bike and is used on the server to identify the bike owned by the player. Do not change or re-arrange these numbers. When a rider changes teams or a new rider replaces a retiring rider the player keeps the bike id they have. For example Alex Marquez moved from Repsol Honda to LCR Honda. Players wo had Alex would continue to have alex and the columns relating to the team he was on changed.
name Name of the bike model as it will appear in the UI
brand Brand of the bike
category Open, Team or Pro Team
team_name The name of the team as it will appear in the game
team_logo The name of the team logo png file used in the UI. These 97x59 png files are located in Assets\Resources\BikesAndRiders\TeamLogo
image_url The name of the UI image of the bikes that appear in the UI . These 512x512 png files are located in Assets\Resources\BikesAndRiders\Bikes. Also identifies the thumnail image for the bike and or bike/rider as used in the Scorecard etc. These 128x80 png files are located in Assets\Resources\BikesAndRiders\Thumbs
first_name First name of the rider, spelled correctly with accenting as necessary
last_name Last name of the rider, spelled correctly with accenting as necessary
rider_image_url The name of the UI image of the riders that appear in the UI . These 512x512 png files are located in Assets\Resources\BikesAndRiders\Riders also identifies the face image (157x89 png) of the rider as used in the Fan World Championship. These assets are located in Assets\Resources\BikesAndRiders\ProRider
number The rider's number. These also identify the number image, 150x150 png files located in Assets\Resources\BikesAndRiders\Numbers
country The country of the rider
flag_image_url The name of the flag image of the rider. I don't know how these are discovered by the UI
abbreviation The abbreviation of the rider as used on the live leaderboard on television. These are used in race to identify the riders ahead and behind you
bike_prefab The name of the bike prefab used by this rider. Generally there is only one bike prefab per team, the exception is generally LCR which tends to have different livery per rider
rider_gloss The color channel to use as a gloss channel in rendering the bikes. 1=red, 2=green, 3=blue. The higher the color value the greater the gloss. By choosing one of the color channels we save a separate channel in the shader for gloss. Remember white is red+green+blue so selecting 1 as the gloss channel would make green and white areas glossy and red and blue areas as matte.
bike_gloss As for the bike gloss. Note that depending on the color scheme of the rider's leathers a different choise of glosschannel may be preferred
rider_pant_leg_in Dianesse suits have the pants over the boots, for Alpinestars the pant leg is tucked in. A variation in the rider model is chosen in game according to this flag to reflect this visual difference.
rider_texture This is the texture used for the rider model in game. Note that there are distinct bike models so each has its own prefab with textures mapped accordingly. For riders there are only two mocels, the pant leg in and pant leg out model. The appropriate texture is then mapped onto the model. The 512x512 3 channel TGA files are located in Assets\Resources\RiderTextures\Body and Assets\Resources\RiderTextures\Helmets Note helmets are always rendered glossy
Coins default price of the bike in coins, can be overridden on the server
Diamonds default price of the bike in diamonds, can be overridden on the server
USD_cents not used
speed actually 'braking' the legacy name 'speed' is used but this refers to the default braking upgrade state when the bike is purchased
speed_max the maximum braking upgrade state
acceleration the acceleration upgrade state upon purchase
acceleration_max the maximum acceleration state for this bike
handling the handling upgrade state upon purchase
handling_max the maximum handling state for this bike
personal_trainer the personal_trainer upgrade state upon purchase
personal_trainer_max the maximum personal_trainer state for this bike
general_manager the general_manager upgrade state upon purchase
general_manager_max the maximum general_manager state for this bike
head_mechanic the head_mechanic upgrade state upon purchase
head_mechanic_max the maximum head_mechanic state for this bike
publicist the publicist upgrade state upon purchase
publicist_max the maximum publicist state for this bike
coinsEconomy The economy_id on the server for this bike, this is where the actual price is determined and subtracted on purchase
diaEconomy The economy_id on the server for this bike, this is where the actual price is determined and subtracted on purchase
sortOrder The sort order of the bikes in the ui when we scroll through pro riders
discountDiamonds no longer used?
discountDiaEconomy economy_id on server for discounted price in diamonds. not sure if still used

3. Migration and merging of riders

On a season update most riders are migrated sucessfully just by changing the BikeInfoTable. e.g.

  • for a A rider moving teams we keep the player owning the same rider, and therefore the same bikeid, and just update the team name etc. an example in 2021 was AlexMarquex moving from Repsol Honda to LCR Honda.

  • for a rider being replaced by another rider we just substitute the new rider for the old.

5. Rider Quality

The rider's arrangement on the grid is randomized but takes into account their 'quality'. This is updated on an annual basis in the file MotoGPGamePlugin.cs. Search for BikeTables.MotoGPSeason == for example here is the relevent code block for 2021:

        else if (BikeTables.MotoGPSeason == 21)// 2021
            MotoGPGamePlugin.Instance.gMaxRiderOrdering = 22;
            // RiderRankedOrder is the default arrangement, they are ranked according to who would be most likely to win, everything else being equal
            MotoGPGamePlugin.Instance.RiderRankedOrder = new int[] { 32, 27, 10, 89, 46, 9, 23, 73, 42, 21, 44, 30, 33, 41, 88, 12, 43, 93, 63, 36, 5, 20 };
        }

        else if (BikeTables.MotoGPSeason == 21)// 2021

The rider numbers are arranged from back to front. E.g. Quatararo (number 20), the current world champion is of the best 'quality' and will most likely appear near the front of the pack each race, but not always.

The value gMaxRiderOrdering also has to match the number of riders. For 2021 it was 22.

7. Bike gloss etc.

The bike shader has available to it a DiffuseMap, a NormalMap and a few extra parameters set in the materials in unity. All maps are provided as tga files and are then compressed by unity. The diffuse map generally a 3 channels for rgb, though an alpha channel can be used for gloss modulation. For three channel maps we set the gloss map as r, g or b. This is controlled in the BikeInfoTable in the parameter bike_gloss, 0,1,2,3 or r,g,b,a respectively. Glossiness is generally using a color channel to same memory, and therefore is not as high quality as providing a separate glossiness map in a, but it provides good enough results.

Getting a bike to look good and closely match the paint of the real bikes depends on balancing all the parameters and one size generally does not fit all. A few examples:

  • the Repsol bike is red, white and yellow, this means that the red channel is maxed out in the diffue for almost all the body, and green is maxed out in the yellow and white areas. For a bright bike like this it is better to dull the diffuse in the shader so there is room for the specular and reflection effects to affect the finished look of the bike.
  • The Suzuki bike has a more 'metalic' looking paint. This is created using the specular effect with lower power.
  • The Red Bull KTM bike has a matte paint so the reflection color is toned down to black
